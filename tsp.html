Traveling Salesman Problem

Written in C++

In this project I found the minimum spanning tree, using Prim's algorithm, and completed the traveling salesman problem for a set of given coordinates. I completed the tsp (traveling salesman problem) in a fast manner (O(n^2) time) using a heuristic called "farthest insertion". In this algorithm, I found the furthest point away from the current point I was looking at, then connected them in the least expensive way possible. I also completed the tsp in the most optimal way using a branch and bound algorithm. In this algorithm I added points to make each possible cycle, but if I deemed a cycle to not be "promising" I would strop exploring that path. In order to determine if a possible cycle was promising, I would add a minimum spanning tree to the current point I was at and determine if this distance was lower than my current best distance or not. This algorithm can be quite slow in the worst case (O(n!) time), but in practice it does much better because many of the possible cycles get marked "not promising" and I don't have to explore them.
